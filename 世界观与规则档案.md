# 世界观与规则档案

适配项目：
`血月降临：只有我看得见规则`

用途：
- 长期记录已出现的规则、怪谈、编号、地点和阶段性判断
- 防止后续写作中规则前后矛盾
- 方便快速查询：某条规则在哪一章出现、在哪一章验证、现在是否已回收

说明：
- 当前已更新到第 40 章
- 以“已正文落地”为准，不以只存在细纲的设定为准

## 总体世界观现状

- 血月降临后，规则怪谈开始侵入现实。
- 江城并非只有一个副本，而是存在多个规则亮点/节点。
- 医院旧区、十三层、307病房、地铁站、公交调度中心之间存在明显连接。
- 这些连接不是随机巧合，更像是被某套“观测/记录/编号”系统串联起来。
- 陆沉至少在这套系统的记录里被标记为 `L-13`。
- 这套系统不只会记录结果，还会预判对象行动趋势，并在极短滞后后完成写实。
- 第30章后已确认：高危亮点不一定是真正入口，真正入口可能藏在被遮蔽的下层死角。
- 第40章后已确认：真正入口后面不是单一副本，而是一整条 `接入 -> 等待 -> 记录 -> 退回 -> 观察` 的归档处理链。
- 前编号对象不会统一消失；一部分只剩识别残片，一部分会以“回声”形式残留在系统缝隙里。
- `L-13` 已从“对象编号”推进到一份 `归档中断 / 接回现实层 / 持续观测` 的未闭合档案。

## 已确认的阶段性判断

### 1. 医院不是起点，但一定是最早爆开的点之一
- 依据：
  - 第10章医院外依旧异常
  - 第16章档案证明十年前旧院区就发生过异常

### 2. 十三层不是凭空鬼楼层
- 依据：
  - 第9章主角判断十三层更像寄生在医院里的规则夹层
  - 第16章查到 `特护3区` 被封、`13` 与 `307` 并列出现

### 3. 307病房和十三层属于同源异常的不同表现
- 依据：
  - 第16章图纸与病区记录把 `13` 和 `307` 并列
- 当前状态：
  - 已形成强判断
  - 但还未拿到终极说明

### 4. 医院、地铁、公交调度中心属于同一套记录系统
- 依据：
  - 第20章调度室中的 `江城主网`
  - 地铁 `L-13`
  - 观测记录
- 当前状态：
  - 已被明确坐实

### 5. 陆沉至少在记录层被标记成 L-13
- 依据：
  - 第16章病区名单 `病人编号：L-13`
  - 第17章完整登记卡 `姓名：陆沉 / 编号：L-13`
  - 第20章观测对象记录
- 当前状态：
  - 硬证据成立
  - 真实身份未揭透

### 6. 城市里的亮点并不处在同一层级
- 依据：
  - 第26章主角第一次明确把亮点看出层次
  - 第27章三处入口有不同诱惑和不同稳定度
- 当前状态：
  - 已被主角阶段性识别
  - 但还未完全命名和坐实

### 7. 节点会动态变化，系统对行动趋势的记录存在极短滞后
- 依据：
  - 第26章现实里的灯先灭，规则视界晚半拍追上
  - 第28章弱点突然疯涨
  - 第29章老张错层时状态更新滞后
  - 第30章高危节点提前出现 `正在准备接入线路` / `通道建立中`
  - 第30章主角利用假动作误导系统成功
- 当前状态：
  - 已被验证
  - 不只是“更新慢半拍”，还包含“先预判、后记录、再执行”的处理流程
  - 是主角后续主动反制系统的重要基础

### 8. 真正入口不一定在最亮的点上，系统会用高危节点当诱饵
- 依据：
  - 第28章原本极弱的点被系统强行灌成高危红点
  - 第30章陆沉借诱饵点让系统错判，逼出隐藏入口
- 当前状态：
  - 已形成强判断
  - 仍需后续验证这种“诱饵点 / 真入口分层”是否普遍存在

### 9. 归档不是简单死亡，而是一套对象补写 / 确认 / 分配流程
- 依据：
  - 第35章等待区出现 `已确认`
  - 第36章登记单出现 `待补写 / 移交模板 / 待分配`
- 当前状态：
  - 已被直接坐实
  - 其核心恐怖在于把活人一步步改写成系统需要的“模板”

### 10. L-13 是一份未闭合档案，陆沉曾被接回现实层持续观测
- 依据：
  - 第36章 `L-13：档案不闭合`
  - 第38章第一页 `接回现实 / 归位失败 / 持续观测`
  - 第40章观察席记录 `归档流程：中断 / 处置：接回现实层`
- 当前状态：
  - 硬证据成立
  - 但“为什么会被接回现实层”仍未揭透

### 11. 这一层系统识别人时更依赖状态栏和流程位，可被反向登记误导
- 依据：
  - 第36章登记单里的 `待补写 / 移交模板 / 待分配`
  - 第39章 `已移交 / 陪同 / 无需复核`
  - 第39章主角反向登记成功
- 当前状态：
  - 已被验证
  - 是主角后续更高阶反制系统的基础

### 12. 未闭合对象会留下回声或残片，不会被彻底清空
- 依据：
  - 第34章 `09` 残片自动认人
  - 第37章 `04号回声`
  - 第40章观察席中的长期占位与确认痕迹
- 当前状态：
  - 已被明确建立
  - 但这些残留形式的边界仍未完全坐实

  - 与陆沉存在硬绑定，但成因未揭透

### 归档
- 首次明确出现：第20章章尾 `13号，欢迎归档`
- 后续加强：
  - 第25章普通人会被写入观测页
  - 第29章名字灰黑化，像被系统注销
  - 第30章真正入口墙上直接出现 `13号，欢迎归档`
  - 第35章等待区出现 `已确认`
  - 第36章出现 `待补写 / 移交模板 / 待分配`
- 当前判断：
  - 不是简单死亡
  - 是对象补写、确认、模板移交和待分配的整套流程
  - 某些对象如果档案未闭合，不会被彻底收完，还可能被接回现实层持续观测
  - 且会对特定编号对象显示出定向接收意味

### 退回格

- 当前判断：
  - 存放未闭合、被退回或没法正式归档的关键首页和底单

### 档案不闭合
- 首次明确出现：第36章 `L-13：档案不闭合`
- 当前判断：
  - 不是简单文件缺页

  - 已被陆沉拔走，等于暂时切断了观察席这一层的稳定锁定
