# 人物设定与关系

适配项目：
`血月降临：只有我看得见规则`

用途：
- 防止长篇连载中人物设定漂移
- 记录人物当前立场、关系变化、已发生关键事件
- 后续写正文时快速查询，避免前后矛盾

说明：
- 目前已更新到前 1 到 40 章正文
- 后续每推进 3 到 5 章，建议更新一次
- 重点不是写人物小传，而是记录“不能写错的信息”

## 主角组

### 陆沉
- 身份：江城第三精神病院夜班护工，主角
- 初始形象：低身份、长期被轻视、寡言、能忍
- 当前能力：
  - `规则视界`：能看见规则、死亡提示、部分危险区域强弱
  - 能力会随着 `成功避死 / 救人 / 带人活下来` 逐步变强
  - 已开始从“看局部死法”升级到“看规则亮点/节点分布”
  - 已开始从“看亮点强弱”升级到“区分不同层级的点”
  - 已确认系统记录与现实变化之间存在极短滞后
  - 已开始利用这种滞后反向试探系统，能用假动作干扰其线路预判
  - 已开始从“看节点强弱”升级到“读流程状态栏 / 分流位 / 接入顺序”
  - 已能利用旧编号残留、状态栏和错误身份做 `反向登记`
- 当前处境：
  - 已成为幸存者队伍实际核心
  - 被十三层和医院旧历史明显针对
  - 镜子怪谈曾给出死法提示
  - 门牌背后全写他的名字
  - 现实旧档案里已出现与他强相关的名字、编号、旧照片
  - 在一套“观测记录/主网记录”里，被标记成 `L-13`
  - 已摸到“先预判、后记录、再执行”的系统缝隙
  - 已确认自己与 `L-13` 存在 `归档中断 / 接回现实层 / 持续观测` 的硬关联

- 对外形象变化：

  - 第35章：先于陆沉听出假叫号的节拍不对
  - 第37章：第一次在身份诱导层把陆沉从错误方向拽回来

  - 被陆沉粗暴拉开，第一次被救

  - 在 `04号回声` 面前分辨“真线索”和“假引导”
  - 在补档区队伍翻面时明确站队，没让陆沉被顺着系统意思留在里面

  - 不是感情线工具人，目前更像“第一副手 + 信息副手”
  - 现在已经不只是情绪支撑或信息补足，也能承担关键执行和身份筛错
- 后续待回收：
  - 她对旧院区、旧档案室还能提供多少信息
  - 她对陆沉的信任是否会进一步升级

- 最近一次出场状态：
