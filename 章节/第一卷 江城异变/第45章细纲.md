# 第 45 章细纲 回来的人

本章定位：
第四十五章的任务，是把 `接回现实层` 这件事从陆沉个人谜团，推进成一条存在过很多次的系统处理方案。但它不能写成蹲在护士站看名单的大说明书，也不能写成“名单一翻，所有真相都清楚了”。最稳的写法，是用一份极其残缺、极其不完整、却足够瘆人的接回名单，让读者第一次真正意识到：被接回来的，不止陆沉一个；回来的人，也不一定还能算“回来”。

本章目标：
- 承接第44章章尾的 `接回名单`
- 坐实 `接回现实层` 不是偶发，而是固定处理方案
- 推进“回来的病人后来怎么样了”这条线
- 继续压老李的普通人恐惧，不让他只是旁边发抖
- 章尾用 `观察席轮值` 把观察者痕迹推进一层

本章核心冲突：
- 主冲突：众人能不能在病区秩序继续运转的情况下，从名单里读出足够值钱的信息，而不被“接回流程”本身盯上
- 次冲突：名单越看越说明这地方不是第一次做这套事，队伍会更难维持“继续查下去还有意义”的信心
- 恐怖来源：残缺名单、被划掉的对象、返送记录、空白床位、仿佛病区还在按名单等人的秩序感

本章回报类型：
- 本章回报重点是：`接回现实不是陆沉特例`

承接开头：
- 紧接第44章章尾
- 护士站抽屉里已经翻出了 `接回名单`
- 众人刚经历一轮空床认人，心理上会对“名单”和“床位”天然形成更强联想

场景 1：这不是名单，是病区工作流

内容要点：
- 开头不要急着逐条读内容，先把名单本身的材质和状态立住。
- 最适合的是：
  - 纸很脆
  - 边角潮烂
  - 很多地方被反复涂黑、划掉、盖章
  - 有的页角还被红笔改过床号
- 这说明它不是一份“归档完成后存放的静态名单”。
- 更像是一份反复在病区现场被使用、被改、被废弃、又被重新抽出来的工作单。
- 周岚这时很适合补一句很值钱的现实判断：
  - 病区如果真的还在照这个名单收人
  - 那每张空床都不是摆设
  - 都是在等名单上的某一格补满

这一场的作用：
- 把名单先落成“仍在运转的流程工具”
- 不是一份单纯的真相说明书

场景 2：回来的不止一个

内容要点：
- 正式读名单时，不能一条条念出来。
- 最适合的是：
  - 名字残缺
  - 床号残缺
  - 有的人只剩对象代号
  - 有的人只剩“接回 / 失败 / 返送”这类状态词
- 最值钱的点不在具体有谁。
- 而在于它非常明确地说明：
  - `接回现实层` 不是陆沉独有
  - 这是这套系统曾经多次尝试过的一种处理方案
- 但不要把名单写成密密麻麻几十条。
- 只要让读者一眼看懂规模：
  - 有人被接回过
  - 有人接回后失败
  - 有人又被返送
  - 还有的人状态干脆空着，像是没来得及写完

这一场的作用：
- 把“陆沉特例”拉成“系统固定方案”
- 同时留住名单本身的神秘感

场景 3：回来之后，不一定还是人

内容要点：
- 这章最瘆人的点不能只是“名单上写了几个人”。
- 必须让读者意识到：
  - 回来不等于安全
  - 回来之后的人，也可能已经不是完整的人
- 最合适的方式是：
  - 在某几条残缺记录边上，出现很短的备注
  - 比如：
    - `夜间脱位`
    - `床外徘徊`
    - `返送前出现替代反应`
    - `语言保留`
- 这些词非常狠。
- 因为它们不用解释，就能让读者自动脑补：
  - 有人回来了
  - 但回来的不是完整活人
  - 有的可能只回来了一部分
- 陆沉会从这些备注里意识到：
  - `接回现实层` 不是救人
  - 更像系统在尝试把某些“还没处理干净的对象”放回现实继续观察

这一场的作用：
- 让接回机制从“会发生”推进到“结果很可怕”
- 为后面回拖、断线、第二接回对象铺路

场景 4：老李的普通人恐惧

内容要点：
- 这一章一定要让老李发挥他的价值。
- 不是让他出错。
- 而是让他把普通人最本能、最现实的恐惧说出来。
- 最合适的是：
  - 他把自己代进名单
  - 想到如果 `09` 后面也会有一条接回 / 返送记录
  - 那他现在根本不在“活着”和“死了”之间
  - 而是在“哪天会不会也被这样写上一笔”
- 这句恐惧很值钱。
- 因为它会把名单从抽象资料重新砸回活人身上。
- 周岚的反应也要跟上：
  - 她不是替老李说话
  - 而是会更怕自己也变成某个床号、某个空白代号

这一场的作用：
- 保住队伍的真实感
- 不让名单这条线飘到“只是主角身世谜”

场景 5：观察席轮值

内容要点：
- 章尾不能只收在“名单很吓人”，必须再往前推进半步。
- 最适合的是：
  - 名单最底部夹着一张薄薄的旧签名条
  - 上面不是对象信息，而是值班信息
- 这张条子不能太完整。
- 最值钱的是一个极淡、极旧、几乎快看不见的章：
  - `观察席轮值`
- 这个词一出来，意义很大：
  - 观察席不是抽象点
  - 至少曾被“某种会轮值的存在”长期使用
- 本章到这里停就够。
- 不要再急着解释“谁轮值”。

章尾钩子：
- 名单最底部夹着一张薄薄的旧签名条。
- 角落里，盖着一个极淡的小章：
  - `观察席轮值`

本章节奏提醒：
- 第45章重点是“名单很吓人”，但不能写成清单说明会
- 每条记录只给一小截，靠状态词和备注吓人，不靠长解释
- 老李必须在这章发挥“普通人恐惧代入”的价值
- 章尾推进到 `观察席轮值` 就够了，不要多讲

本章伏笔：
- 接回现实层是重复发生过的固定处理方案
- 接回来的人未必还是完整活人
- 特护3区和观察席之间存在长期交叉工作关系
- 后面一定还有更旧的值班记录角能继续往里挖

本章读者爽点：
- “接回现实”终于从主角特例变成系统方案
- 名单上的几个状态词会非常有画面联想
- `观察席轮值` 这个章尾会让人自然想看第46章
