# 第 32 章细纲 过闸

本章定位：
第三十二章要完成封闭换乘区的第一次硬验证。第31章已经把“接收前厅”和规则气质立住了，这一章就不能再只看环境、只读规则，而要让读者真正明白：这里的闸机不是放人通行的，它是在确认你是谁、该往哪边去。恐怖重点不是闸机后面有什么，而是你一旦被它认错，整个人会先被“分出去”半截。

本章目标：
- 承接第31章章尾的 `A-13，待接入`
- 完成封闭换乘区第一轮硬验证
- 明确闸机的本质不是通行设备，而是“分类口”
- 让陆沉第一次判断出：这里不只看行动，还看顺序、距离、站位
- 安排一个普通幸存者差点被“分出去”的现场打击
- 让林晚继续承担现实结构辅助
- 章尾亮出 `01 / 04 / 09 / 13` 四个编号屏，正式打开“前编号对象”这条线

本章核心冲突：
- 主冲突：面对已经亮起的接入闸机，众人到底该不该靠近、该怎么过去
- 次冲突：系统明显在点陆沉，队伍会本能想把风险往他一个人身上推
- 恐怖来源：单独亮起的旧闸机、会念名字的播报、地面影子和玻璃倒影先一步被吞、被分类的压迫感

本章回报类型：
- 本章回报重点是：`换乘闸机第一次硬验证`

承接开头：
- 紧接第31章结尾
- 最深处那道旧闸机已经亮起：
  - `A-13，待接入。`
- 众人已经知道这里的规则重点是：别被排队、别被认领、别被并排分流

场景 1：不是所有闸机都能过

内容要点：
- 开头先把闸机区整体亮出来。
- 众人真正走近后会发现：
  - 那不是一台闸机
  - 而是一整排早该废掉的旧分流口
- 但诡异的地方在于：
  - 只有 `A-13` 那一口亮着
  - 其余闸机都灰着，却像随时也能被叫醒
  - 每一道闸机上方都留着老式编号位，像曾经各自对应不同对象
- 陆沉第一反应不能是“终于找到路了”。
- 更合适的是：
  - 他立刻意识到这不是给所有人开的门
  - 而是一道专门冲着某个目标亮起的接入口
- 老李会很自然地冒出一句危险念头：
  - 既然它叫的是你
  - 那是不是你过去，其他人就能不跟着受牵连
- 这句台词很重要。
- 因为它第一次把“把风险往陆沉一个人身上推”的人性苗头露出来。

这一场的作用：
- 把“闸机区”从单一道具升级成一套分类系统
- 埋下队伍后面更深的人性压力

场景 2：陆沉的第一判断

内容要点：
- 陆沉不能被 `A-13` 这一下带着走。
- 他会先用规则视界看整排闸机：
  - 亮着的不一定最安全
  - 血光最稳的也不一定是真通道
- 他这时要得到一个很关键的初步判断：
  - 这里的闸机不是“开了就能过”
  - 而是“亮了就在准备接”
- 林晚在这一场继续补现实结构：
  - 她会发现每道闸机后面的缓冲区长度都不一样
  - 有的像正常排队口
  - 有的短得根本不像给活人通过
- 这能支撑陆沉进一步判断：
  - 有的口子不是拿来让人走的
  - 更像拿来把人“截进去”的
- 他会下一个非常明确的命令：
  - 不许并排
  - 不许抢先
  - 叫到谁都不准回头看

这一场的作用：
- 让陆沉的判断开始从“看危险”升级成“读流程”
- 给后面的硬验证提前立好秩序

场景 3：名字被叫到

内容要点：
- 众人刚按顺序靠近，最危险的事就发生了。
- 最适合的方式是：
  - 队伍里一个普通幸存者先被叫到了名字
  - 广播不是从头顶来，而是像从他身后、从闸机玻璃里冒出来
- 规则已经说了：
  - 听见闸机播报自己的名字，不要回头
- 但人性上最难忍的就是这种“有人就在背后叫你”。
- 那个人会下意识偏头。
- 这一偏不能太夸张，越轻越吓人。
- 因为就在他偏头的同时：
  - 面前闸机玻璃里的倒影没有跟着一起偏
  - 反而先一步直直走了进去
- 同一时间，他脚下的影子也像被什么东西拽了一把，硬生生越过了闸线半截。
- 这里的恐怖感一定要落在“人还在，但一部分已经先被分过去了”。

这一场的作用：
- 完成第一条高价值规则的现场硬验证
- 让“被分出去”这个概念第一次有了具体画面

场景 4：不能并排救人

内容要点：
- 那个被点名的人一出事，旁边一定会有人下意识去拉。
- 但规则里还有一句更危险的：
  - 任何两个人，不要并排通过同一道分流闸口
- 这里最适合安排：
  - 另一个幸存者本能想冲上去拽人
  - 差一点就和对方肩并肩撞进同一道闸口
- 陆沉必须在这一瞬间喝住。
- 最合适的操作是：
  - 不是往前扑
  - 而是用手里能勾到的旧伸缩带、废指示牌，先把两个人隔开
- 林晚可以在这一场承担执行辅助：
  - 她负责把旁边的人往后拽
  - 让陆沉腾出手处理前面那个人的“半截影子”
- 最终结果不能写成轻松无伤：
  - 人要被勉强拖回来
  - 但他的影子短了一小截，或在接下来一段时间里对光的反应不对
- 这样既有硬打击，又不至于直接写死。

这一场的作用：
- 证明第二条规则同样是硬的
- 让读者感受到这里“连救人姿势都必须符合流程”

场景 5：闸机真正认的不是人，是身份

内容要点：
- 把人拖回来后，陆沉会盯着那道闸机看出一个新门道。
- 他会注意到：
  - 闸机亮起之前，先播报名字
  - 影子和倒影先动
  - 最后才轮到人的身体被牵扯
- 这说明闸机真正认的，不只是“一个活人站在这里”。
- 它更像是在先确认：
  - 你是谁
  - 你是不是它要接的那个人
  - 你有没有被分对
- 这一章最关键的成长判断可以落在这里：
  - 真正危险的不是哪道闸机亮了
  - 而是它认你是谁
- 林晚可以补一句非常值钱的现实判断：
  - 正常闸机验证的是车票
  - 这里验证的是人

这一场的作用：
- 把本章主题从“过闸惊险”拔高到“身份分类”
- 为第33章的编号等待区做逻辑承接

场景 6：四块屏幕同时亮起

内容要点：
- 陆沉不会贸然走进 `A-13` 那道亮口。
- 他会选一条暂时最不“主动”的灰闸，带众人按单列顺序小心通过。
- 穿过去后，后面不该立刻出现车轨或正常换乘厅。
- 最适合的是：
  - 一整排封闭站台屏幕同时亮起
  - 玻璃门后面根本没有列车轨道
  - 只有被封死的黑墙
- 屏幕最开始还像正常站台一样跳站名。
- 但只闪了两下，所有字同时扭曲、塌缩，最后只剩四个编号：
  - `01`
  - `04`
  - `09`
  - `13`
- 这一幕要收得很硬。
- 因为它会立刻把“陆沉不是唯一对象”这条线提前咬出来。

章尾钩子：
- 过闸后的几块封闭站台屏幕，同时亮起。
- 上面没有站名，只有四个编号：
  - `01`
  - `04`
  - `09`
  - `13`

本章节奏提醒：
- 第32章必须给硬验证，不能还停留在“看环境、念规则”
- 恐怖重点不是有人被追，而是“身体还在，人已经先被分类了半截”
- 这一章要让读者明确理解：闸机验证的不是车票，不是路线，而是身份归属
- 章尾不要解释 `01 / 04 / 09 / 13`，先炸，再留到后面吃

本章伏笔：
- 闸机会先叫名字，再拖影子 / 倒影，说明分类先于实体接收
- `A-13` 明显是专门对陆沉亮起的接入口
- 被拽回来的幸存者后面还能继续承担“轻度被分过一次”的余波
- 四个编号说明在陆沉之前，系统里至少还有别的对象

本章读者爽点：
- 封闭换乘区第一轮硬验证成立
- “倒影先走进去、影子被拽过去半截”的画面很有记忆点
- 章尾四编号亮屏，能把读者追更欲直接拉起来
